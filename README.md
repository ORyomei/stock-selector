# Stock Selector — AI 駆動型 株式売買判断システム

## 概要

ニュース・株価・SNS のつぶやき等の情報を AI が総合的に分析し、**どの銘柄を・いつ・どの方向（買い/売り）で動くべきか**を推論・提案するシステム。

従来の「アプリケーション」ではなく、**GitHub Copilot (Agent) がこのワークスペースを起点として自律的に動作する**形式を採る。ユーザーが指示を出すと、Copilot が MCP・CLI・Web API 等を駆使して情報収集・分析・判断を行い、結果を報告する。

```
│    │   ├─ ニュース取得（RSS / News API）           │

│    ├─ 分析レイヤー                                │
│    │   ├─ センチメント分析                         │

│    │   └─ 総合スコアリング                         │

│        ├─ diary/ への判断記録                      │
│        └─ ポートフォリオ追跡                       │
└─────────────────────────────────────────────────┘
```

## 要件定義

### 1. 情報収集

| ソース | 目的 | 取得方法 |
|--------|------|----------|
| **株価** | 現在値・過去の値動き・出来高 | `yfinance` (Python) または無料 API |
| **ニュース** | 企業・セクター・マクロ経済のイベント | RSS フィード / News API / Web スクレイピング |
| **SNS (X 等)** | 市場センチメント・リアルタイムの話題 | X API (v2) / 代替手段 |
| **経済指標** | 金利・為替・CPI 等 | FRED API / 公開データ |

### 2. 分析

- **センチメント分析**: ニュース・SNS テキストの感情極性（ポジティブ/ネガティブ/ニュートラル）を判定
- **テクニカル分析**: 移動平均 (SMA/EMA)、RSI、MACD、ボリンジャーバンド等を算出
- **ファンダメンタル概要**: PER、PBR、配当利回り等の基本指標
- **総合スコアリング**: 上記を統合し、各銘柄に対する売買シグナル強度を算出

### 3. 判断出力

- **対象市場**: 日本株（東証）および米国株（NYSE/NASDAQ）
- **推奨アクション**: 買い / 売り / ホールド / 様子見
- **推奨タイムスパン**: デイトレ～スイング（数日）～中期（数週間〜数ヶ月）
- **確信度**: 高 / 中 / 低
- **根拠**: 判断に至った主要因を箇条書きで示す

### 4. 記録

- 判断のたびに `diary/YYYY-MM-DD.md` へ記録を追記
- 過去の判断と実際の結果を対比できるように構造化
- ポートフォリオ状態を `portfolio.json` で追跡（仮想）

### 5. 運用フロー

1. ユーザーがこのワークスペースを開き、Copilot Chat で指示を出す
   - 例: 「今日の市場状況を分析して」「〇〇の銘柄を調べて」「ポートフォリオを見直して」
2. Copilot が情報を収集し、分析スクリプトを実行
3. 結果と売買判断を Chat 上で報告
4. 判断内容を `diary/` に記録
5. 必要に応じてポートフォリオを更新

## 制約・注意事項

- **投資助言ではない**: 本システムの出力は参考情報であり、投資判断の最終責任はユーザーにある
- **リアルタイム性の限界**: 無料 API のレート制限により、リアルタイムデータには遅延がある場合がある
- **API キーの管理**: `.env` ファイルで管理し、Git にコミットしない（`.gitignore` に含める）
- **自動売買は行わない**: あくまで「判断の提案」であり、実際の発注は行わない
